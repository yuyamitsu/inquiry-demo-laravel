# 問い合わせ管理Demo Laravel版 設計資料

## 目的

静的Demoで作成した問い合わせ管理機能を、Laravel + DB構成に置き換える。

## 画面構成

| 画面 | URL | 内容 |
| --- | --- | --- |
| 問い合わせフォーム | / | 利用者が問い合わせを登録する |
| 管理者一覧 | /admin/inquiries | 問い合わせを一覧・検索・絞り込み表示する |
| 管理者詳細 | /admin/inquiries/{inquiry} | 問い合わせ詳細確認、ステータス更新、返答保存を行う |

## 機能一覧

| 機能 | 内容 |
| --- | --- |
| 問い合わせ登録 | 入力内容をバリデーション後、DBへ保存 |
| 一覧表示 | 登録済み問い合わせを新しい順に表示 |
| 詳細表示 | 問い合わせ内容と管理者返答欄を表示 |
| ステータス更新 | 未対応・対応中・回答済み・クローズを管理 |
| 返答保存 | 管理者の返答内容を保存 |
| 検索 | 件名・名前・メール・本文を対象に検索 |
| 絞り込み | ステータス・カテゴリで絞り込み |
| 削除 | 不要な問い合わせを削除 |

## ステータス設計

| ステータス | 意味 |
| --- | --- |
| 未対応 | 登録直後の初期状態 |
| 対応中 | 管理者が確認・対応している状態 |
| 回答済み | 利用者への回答が完了した状態 |
| クローズ | 対応完了として終了した状態 |

## 処理フロー

### 問い合わせ登録

利用者がフォームに入力
↓
POST /inquiries
↓
InquiryController@store
↓
バリデーション
↓
Inquiry::create()
↓
inquiriesテーブルへ保存
↓
管理者一覧へ遷移
