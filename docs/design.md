# 問い合わせ管理Demo Laravel版 設計資料

## 1. 目的

本資料は、問い合わせ管理Demo Laravel版の画面構成、機能構成、権限制御、処理フローを整理するための設計資料です。

本Demoは、静的Demoで作成した問い合わせ管理機能を、Laravel + DB構成に置き換えたものです。

現在は単なる問い合わせフォームではなく、問い合わせ1件を「チケット」として扱い、以下の機能を持つ中規模向けの問い合わせ・チケット管理Demoとして設計しています。

```text
・問い合わせ登録
・問い合わせ一覧
・問い合わせ詳細
・コメントスレッド
・担当者管理
・優先度管理
・対応期限管理
・ステータス管理
・変更履歴
・ユーザー管理
・パスワード変更
・仮パスワード再設定
・ナレッジ管理
・権限制御
```

---

## 2. システム概要

### システム名

```text
CoreviaDesk
```

### 概要

CoreviaDesk は、問い合わせをチケットとして管理し、対応状況、担当者、優先度、期限、コメント履歴、変更履歴を一元管理するDemoシステムです。

また、解決した問い合わせをナレッジ記事として保存することで、問い合わせ対応で得た情報を再利用できる構成にしています。

---

## 3. 利用者種別

本システムでは、`users.role` によって権限を分けます。

| role  | 名称         | 主な役割                          |
| ----- | ---------- | ----------------------------- |
| admin | 管理者        | 全体管理、ユーザー管理、問い合わせ削除、仮パスワード再設定 |
| staff | スタッフ / 担当者 | 問い合わせ対応、コメント投稿、ナレッジ管理         |
| user  | 一般ユーザー     | 問い合わせ登録、自分の問い合わせ確認、コメント投稿     |

## 5. 画面構成

### 認証画面

| 画面         | URL         | 内容                 |
| ---------- | ----------- | ------------------ |
| ログイン画面     | `/login`    | メールアドレスとパスワードでログイン |
| 一般ユーザー登録画面 | `/register` | 一般ユーザーとして新規登録      |
| ログアウト      | `/logout`   | ログアウト処理            |

一般ユーザー登録画面から登録したユーザーの `role` は必ず `user` になります。

管理者やスタッフは、管理者向けユーザー作成画面から作成します。

| 画面        | URL                             | 内容                  |

## 6. 主な機能一覧

| 機能         | 内容                            |
| ---------- | ----------------------------- |
| 認証         | ログイン、ログアウト、一般ユーザー登録           |

| 自分の問い合わせ確認 | 一般ユーザーは自分の問い合わせのみ確認           |
| 管理側一覧      | admin / staff が全問い合わせを一覧確認    |
| 管理側詳細      | 問い合わせ内容、コメント、変更履歴を確認          |
| ステータス管理    | 未対応、対応中、回答済み、クローズを管理          |
| 担当者管理      | admin / staff を問い合わせ担当者として設定  |
| 優先度管理      | 低、中、高、緊急を設定                   |
| 対応期限管理     | 問い合わせごとの対応期限を設定               |
| コメントスレッド   | 問い合わせごとに複数コメントを保存             |
| 変更履歴       | 管理項目の変更前後を保存                  |
| 検索         | 件名、名前、メール、本文を対象に検索            |
| 絞り込み       | ステータス、カテゴリ、担当者、優先度、期限状態で絞り込み  |
| ユーザー管理     | adminがユーザー一覧、詳細を確認            |
| ユーザー作成     | adminがadmin / staff / userを作成 |
| パスワード変更    | ログイン中ユーザー自身がパスワードを変更          |
| 仮パスワード再設定  | adminが対象ユーザーの仮パスワードを再設定       |
| ナレッジ作成     | 問い合わせからナレッジ記事を作成              |
| ナレッジ管理     | ナレッジ記事の一覧、詳細、編集、公開状態管理        |
| パンくず       | 現在の画面位置と戻り先を表示                |
| ユーザーメニュー   | ヘッダーにログインユーザー名と関連メニューを表示      |

## 7. ステータス設計

| ステータス | 意味                     |
| ----- | ---------------------- |
| 未対応   | まだ対応していない状態            |
| 対応中   | 管理者またはスタッフが確認・対応している状態 |
| 回答済み  | 利用者への回答が完了した状態         |
| クローズ  | 対応完了として終了した状態          |

---

## 8. 優先度設計

| 優先度 | 意味             |
| --- | -------------- |
| 低   | 急ぎではない問い合わせ    |
| 中   | 通常対応の問い合わせ     |
| 高   | 優先して対応したい問い合わせ |
| 緊急  | 早急な対応が必要な問い合わせ |

---

## 9. 期限状態

| 条件    | 内容                         |
| ----- | -------------------------- |
| 期限切れ  | 対応期限が今日より前、かつクローズではない問い合わせ |
| 今日まで  | 対応期限が今日の問い合わせ              |
| 期限未設定 | 対応期限が設定されていない問い合わせ         |

## 10. DB設計概要

主に以下のテーブルを使用します。

```text
users
inquiries
inquiry_comments
inquiry_logs
knowledge_articles
```

### users

ユーザー情報を管理します。

```text
・ログイン情報
・権限 role
・問い合わせ登録者
・問い合わせ担当者
・コメント投稿者
・ナレッジ作成者
```

### inquiries

問い合わせ本体を管理します。

```text
・問い合わせタイトル
・カテゴリ
・本文
・登録者
・担当者
・ステータス
・優先度
・対応期限
```

### inquiry_comments

問い合わせごとのコメントを管理します。

```text
・問い合わせID
・投稿者ID
・コメント本文
・投稿日時
```

### inquiry_logs

問い合わせの管理項目変更履歴を管理します。

```text
・問い合わせID
・更新者ID
・変更項目
・変更前の値
・変更後の値
・履歴メッセージ
```

### knowledge_articles

問い合わせから作成したナレッジ記事を管理します。

```text
・元問い合わせID
・作成者ID
・タイトル
・カテゴリ
・本文
・公開状態
```

## 11. 主なリレーション

| リレーション                                | 内容                            |
| ------------------------------------- | ----------------------------- |
| users 1 - 多 inquiries                 | ユーザーは複数の問い合わせを登録できる           |
| users 1 - 多 inquiries                 | admin / staff は複数の問い合わせを担当できる |
| inquiries 1 - 多 inquiry_comments      | 問い合わせ1件に複数コメントを持てる            |
| users 1 - 多 inquiry_comments          | ユーザーは複数コメントを投稿できる             |
| inquiries 1 - 多 inquiry_logs          | 問い合わせ1件に複数の変更履歴を持てる           |
| users 1 - 多 inquiry_logs              | ユーザーは複数の変更操作を行える              |
| inquiries 1 - 0..1 knowledge_articles | 問い合わせ1件から0件または1件のナレッジ記事を作成できる |
| users 1 - 多 knowledge_articles        | ユーザーは複数のナレッジ記事を作成できる          |

## 12. 処理フロー

### 12.1 ログイン

```text
ログイン画面
↓
POST /login
↓
AuthController@login
↓
認証
↓
role 判定
↓
admin / staff は /admin/inquiries へ
user は /my/inquiries へ
```

---

### 12.2 一般ユーザー登録

```text
ユーザー登録画面
↓
POST /register
↓
AuthController@register
↓
バリデーション
↓
role = user で users に保存
↓
自動ログイン
↓
/my/inquiries へ遷移
```

---

### 12.3 問い合わせ登録

```text
問い合わせフォーム
↓
POST /inquiries
↓
InquiryController@store
↓
バリデーション
↓
user_id にログイン中ユーザーIDを保存
↓
status = 未対応 で保存
↓
inquiries テーブルへ保存
↓
admin / staff は /admin/inquiries へ
user は /my/inquiries/{inquiry} へ
```

---

### 12.4 管理側問い合わせ一覧

```text
GET /admin/inquiries
↓
auth middleware
↓
admin middleware（admin / staff を許可）
↓
InquiryController@index
↓
検索条件を取得
↓
DB検索
↓
件数サマリー取得
↓
admin/inquiries/index.blade.php を表示
```

---

### 12.5 管理項目更新

```text
PUT /admin/inquiries/{inquiry}
↓
InquiryController@update
↓
admin / staff か確認
↓
バリデーション
↓
変更前の値を保持
↓
inquiries を更新
↓
inquiry_logs に履歴を保存
↓
詳細画面へ戻る
```

---

### 12.6 コメント投稿

```text
POST /inquiries/{inquiry}/comments
↓
InquiryController@storeComment
↓
ログインユーザーが投稿可能か確認
↓
バリデーション
↓
inquiry_comments に保存
↓
直前の詳細画面へ戻る
```

---

### 12.7 ユーザー作成

```text

↓

↓
admin のみ許可
↓

↓

↓

↓

```

### 12.8 仮パスワード再設定

## 13. ナレッジ管理設計

ナレッジ管理は、問い合わせ対応で得た内容を再利用するための機能です。

問い合わせ詳細画面から「ナレッジ化」することで、問い合わせ本文とコメント履歴をもとにナレッジ記事を作成します。

### ナレッジ本文の初期構成

```text
【問い合わせ内容】
問い合わせ本文

【対応内容】
コメント履歴

【解決方法】
※ 最終的な解決方法を記入

【補足】
※ 再発時の注意点や関連情報を記入
```

### 公開状態

| 状態  | 意味                |
| --- | ----------------- |
| 公開  | ナレッジとして公開する記事     |
| 下書き | 作成途中、またはまだ公開しない記事 |

## 14. 画面導線設計

### ヘッダー

ログイン中は、ヘッダー右側にログインユーザー名を表示します。

ユーザー名にホバーすると、ユーザーメニューを表示します。

メニュー内容は権限によって分岐します。

| メニュー    | admin | staff | user |
| ------- | :---: | :---: | :--: |
| ナレッジ    |   ○   |   ○   |   ×  |
| ユーザー管理  |   ○   |   ×   |   ×  |
| パスワード変更 |   ○   |   ○   |   ○  |
| ログアウト   |   ○   |   ○   |   ○  |

### パンくず

各画面にはパンくずを表示し、現在位置と戻り先を分かりやすくします。

例：

```text
問い合わせ一覧 ＞ 問い合わせ詳細 #1 ＞ ナレッジ作成
```
